How does javascript work?

Javascript is one of the scripting language that is single threaded. This scripting language solves issues one by one from the starting of the codes to end. Javascript is being used for frontend and also for backend development. This is not a compiled language as other programming languages. It uses 'V8 engine' from chrome browser for compiling the codes to get the output.

What does javascript event loop do?

Javascript is itself a single threaded language. When a long function of processes is called, it takes a long time to execute and finish. Till then, the other lines of code just waits for it to finish. For fixing this issue, the browser APIs works as multi thread and when the event loop works such as async fetch url, it uses empty stack if it is not busy. That is what the event loop do to solves the slow processing issue.

What is the difference between local storage and session storage

There are some differences between local storage and session storage which are given below,

  1. Local storage do not expires automatically, where session storage does after browser is exited.
  2. Local storage is data persistent but session storage is not.
  3. Local storage is not secured as much as session storage.
  4. Local storage might get hacked but session storage can not because if user leaves the tab or the browser, the data gets destroyed.
Others such as storage size, read-write access, used in client side are all the same for both local storage and session storage.
